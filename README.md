# Victoria 3 Translator (DeepSeek V4 Pro)

DeepSeek V4 Pro を使って Victoria 3 MOD の `.yml` ローカライズファイル群を
任意の言語ペア間で翻訳するシンプルな Tkinter GUI ツール。

本プロジェクトは、Paradoxゲームの翻訳自動化の先駆者である [Project Remis](https://github.com/Drlinglong/Remis) (Paradox Mod Localization Factory) の設計思想に深くインスパイアされており、その素晴らしい成果に敬意を表して開発されました。

## 特徴

- **DeepSeek V4 Pro & OpenRouter 対応** — OpenAI互換API（`api.deepseek.com`）や OpenRouter 経由の多様なモデルに対応
- **任意の言語ペア** — English / 简体中文 / 日本語 / 한국어 / Français / Deutsch /
  Español / Русский / Polski / Português do Brasil / Türkçe の 11 言語から選択可能
- **並列翻訳** — `ThreadPoolExecutor` で複数バッチを同時実行（同一ファイル内でも分割して並列処理）
- **詳細なエラー＆レート制限コントロール** — APIの429エラーや、パース失敗などの一般エラー時の動作を個別に設定可能
- **キャッシュ + 再開** — SQLite に翻訳済みペアを保存。中断 → 再実行で未翻訳分のみ処理
- **安全な出力** — 入力を破壊せず `output/<任意の出力名>/` に書き出し
- **Vic3 特殊トークンの保持** — `[ROOT.GetCountry.GetName]`, `$concept$`, `#bold...#!`,
  `\n` などをそのまま保持するようプロンプトで指示

## 必要環境

- Python 3.10 以上
- DeepSeek API Key または OpenRouter API Key

## セットアップ

1. **`setup.bat` を右クリックして「管理者として実行」**
   - （またはそのままダブルクリックすると、自動で管理者権限への昇格プロンプトが表示されます）
2. 依存パッケージ (`openai`, `tqdm` など) が自動でインストールされます。
3. **API Key の設定**:
   - コマンドプロンプト上で DeepSeek/OpenRouter API Key を入力すると、Windowsの環境変数としてシステムに保存されます。
   - スキップして、後から GUI 上で入力・保存することも可能です。

## 使い方

1. `run.bat` をダブルクリックして GUI を起動します。
2. GUI が開いたら：
   - **API Key**: `setup.bat` で設定した場合は自動で入力されています。
   - **ソース言語** と **ターゲット言語** を選択（例: English → 日本語）
   - **MODフォルダ** に `localization` フォルダのパスを指定
     （デフォルトで同梱の `./localization/` が設定済み）
   - 必要に応じて **MODテーマ** に概要を英語で入力
     （例：`Cold War expansion mod for Victoria 3`）
   - **オプション設定**:
     - **並列数**: 同時にいくつの翻訳バッチを走らせるか (デフォルト: 8)
     - **バッチサイズ**: 1回のAPIリクエストに含めるテキストの数 (デフォルト: 40)
     - **推論レベル / 思考モード**: OpenRouter使用時は `reasoning_effort` (none/low/medium/high) を設定可能。DeepSeek使用時は高品質な「思考モード」のON/OFFが可能。
     - **レート制限**: 429 (Too Many Requests) エラー時の動作を選択 (backoff/retry/stop/ignore)。「リトライ数」で429発生時の再試行回数を設定します。
     - **エラー処理**: 429以外のパースエラーや通信エラー発生時の動作を選択 (retry/stop/ignore)。「エラーリトライ」で各バッチ分割時の最大再試行回数を独立して設定できます。詳細は後述。
     - **未翻訳箇所のみ抽出 (混合ファイル対応)**:
       - 既存のターゲット言語ファイルがある場合、キー単位で比較し「すでに翻訳済み」と判定できる文は再翻訳せず保持します。
       - 一部だけ未翻訳のファイルを再実行したいときに有効です。
       - 初回のフル翻訳時は **OFF**、再実行で差分だけ詰めたいときは **ON** を推奨します。
   - **出力先** と **出力フォルダ名(必須)** を指定
     - 既に存在するフォルダ名は指定できません（重複禁止）
3. **翻訳開始** を押す
4. 進捗バーとログで状態を確認。途中で **キャンセル** も可能
5. 完了後、出力先は `output/<指定した出力フォルダ名>/` 配下

## レート制限 (429エラー) とエラー時の動作ポリシー

APIの利用制限に達した際（429）や、その他のパースエラー・通信エラーが発生した際の動作を柔軟に選べます：

### 429 (レート制限) エラー時のポリシー
- **`backoff` (推奨)**:
  自動的に並行ワーカー数（並列数）を半分に減らし、APIへの負荷を下げながら翻訳を続行します。安定して処理を進めたい場合に最適です。
- **`retry`**:
  失敗したバッチを待機して再試行します。ワーカー数は維持しつつ時間を空けて再送したい場合に有効です。429 が続く場合は並列数を下げて併用してください。
- **`ignore`**:
  エラーを無視し、失敗したテキストは原文のままスキップして次のバッチに進みます。とにかく最後まで止まらずに処理を終わらせたい場合に使用します。
- **`stop`**:
  429エラーが出た時点で即座に翻訳タスク全体をキャンセルします。手動で制限解除を待ちたい場合に有効です。

### 429 以外のエラー時 (エラー処理) のポリシー
- **`retry` (推奨)**:
  パースエラー等が発生したバッチを半分に分割し、問題の文字列を特定しながら再帰的に再試行します。成功する文字列を最大限救出できます。
- **`ignore`**:
  エラーが発生したバッチ全体を無視し、原文のままスキップします（分割や再試行を行わないため、最速でエラーをやり過ごせます）。
- **`stop`**:
  予期せぬエラーが出た時点で即座に翻訳タスク全体をキャンセルします。

## 出力構造

例：English → Japanese の場合
```
output/
└─ english_to_japanese/
   ├─ 0_general_l_japanese.yml
   ├─ CWN·Journal/
   │   ├─ cwn_je_japan_l_japanese.yml
   │   └─ ...
   └─ replace/
       └─ japanese/
           └─ ...
```

Vic3 MOD の `localization/` フォルダにそのままコピーすれば反映されます。

## キャッシュの扱い

- SQLite ファイル: `cache/translation_cache.sqlite`
- 同じソース文字列を別ファイルで見かけても、API は一度しか呼ばれない
- 途中で中断しても次回起動時にキャッシュヒットで即復帰
- **キャッシュをクリア** ボタンで全削除可能

## 設定ファイル

- `config/settings.json` — 最後の実行時の言語・パス・モデル設定が保存される
- API キーは「保存」ボタン押下時のみ書き込まれる（平文）
- 環境変数 `DEEPSEEK_API_KEY` が設定されていれば起動時に自動取得

## トラブルシューティング

| 症状 | 対策 |
|---|---|
| `The 'openai' package is not installed` | `setup.bat` を管理者権限で実行して依存関係をインストールしてください |
| レート制限 `429` が出る | レート制限ポリシーを `backoff` または `retry` にし、初期の並列数を下げてください |
| 出力がほぼ原文のままになる | `ignore` になっていないか確認し、`backoff` / `retry` を選択して再実行。必要ならキャッシュをクリアしてください |
| 翻訳後にゲームが文字化け | UTF-8 BOM 付きで出力しているか確認（本ツールは `utf-8-sig` 固定） |
| 特殊トークンが訳されてしまう | `MODテーマ` を詳しく書く、または問題ファイルだけキャッシュクリアして再実行 |
